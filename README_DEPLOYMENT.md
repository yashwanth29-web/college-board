## 🎯 Deployment Steps (30 minutes)

### 1️⃣ Prepare Code (5 min)
```bash
chmod +x backend/build.sh
git add .
git commit -m "Prepare for deployment"
git push origin main
```

### 2️⃣ Deploy Backend to Render (10 min)
- Go to render.com
- Create Web Service
- Connect GitHub repo
- Configure environment variables
- Deploy and get URL

### 3️⃣ Deploy Frontend to Vercel (5 min)
- Update `environment.prod.ts` with backend URL
- Go to vercel.com
- Import GitHub repo
- Deploy and get URL

### 4️⃣ Update CORS (3 min)
- Go back to Render
- Update `CORS_ALLOWED_ORIGINS` with Vercel URL
- Update `ALLOWED_HOSTS`
- Save and redeploy

### 5️⃣ Test Application (5 min)
- Visit Vercel URL
- Test login (admin & student)
- Verify all features work

## 💰 Cost

### Free Tier (Perfect for Testing)
- **Vercel**: Free forever
- **Render**: Free (sleeps after 15min)
- **Neon**: Free (0.5GB storage)
- **Total**: $0/month

### Production Tier (Recommended for Live)
- **Vercel**: Free
- **Render**: $7/month (no sleep)
- **Neon**: $19/month (more resources)
- **Total**: $26/month

---

## ⚡ Performance

### Free Tier
- First load: 30-60 seconds (Render wakes up)
- After wake: 1-3 seconds
- Sleeps after 15min idle

### Paid Tier
- First load: 1-3 seconds
- Always active
- No sleep

## 🐛 Common Issues & Solutions

### Issue: CORS Error
**Solution**: Update `CORS_ALLOWED_ORIGINS` in Render with exact Vercel URL (including `https://`)

### Issue: Backend Slow
**Solution**: Free tier sleeps. Upgrade to $7/month for always-on.

### Issue: Build Fails
**Solution**: Check `build.sh` is executable: `chmod +x backend/build.sh`

### Issue: Database Connection Error
**Solution**: Verify `DATABASE_URL` in Render matches your Neon URL
