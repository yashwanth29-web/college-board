# Pre-Deployment Checklist

Complete these steps before deploying to Vercel and Render.

---

## ✅ Code Preparation

### 1. Make build.sh Executable
```bash
chmod +x backend/build.sh
```

### 2. Verify Environment Files Exist
- [ ] `frontend/src/environments/environment.ts` (development)
- [ ] `frontend/src/environments/environment.prod.ts` (production)
- [ ] `backend/.env.example` (template)

### 3. Update Production API URL
Edit `frontend/src/environments/environment.prod.ts`:
```typescript
export const environment = {
  production: true,
  apiUrl: 'https://YOUR-BACKEND-URL.onrender.com/api'
};
```
**Note**: You'll update this with your actual Render URL after backend deployment.

### 4. Verify Dependencies
```bash
# Backend
cd backend
pip install -r requirements.txt

# Frontend
cd frontend
npm install
```

### 5. Test Local Build
```bash
# Backend
cd backend
python manage.py collectstatic --no-input
python manage.py migrate

# Frontend
cd frontend
npm run build:prod
```

---

## 🔒 Security Check

### Backend Security
- [ ] `DEBUG=False` in production
- [ ] Strong `SECRET_KEY` (will be generated by Render)
- [ ] `ALLOWED_HOSTS` configured
- [ ] `CORS_ALLOWED_ORIGINS` configured
- [ ] No hardcoded credentials in code
- [ ] `.env` file in `.gitignore`

### Frontend Security
- [ ] No API keys in code
- [ ] Environment variables used for API URL
- [ ] No sensitive data in localStorage

---

## 📦 Git Repository

### 1. Verify .gitignore
```bash
# These should NOT be in git:
.env
backend/.env
frontend/.env
node_modules/
venv/
__pycache__/
dist/
```

### 2. Commit All Changes
```bash
git status
git add .
git commit -m "Prepare for deployment"
```

### 3. Push to GitHub
```bash
git push origin main
```

## 📝 Deployment Order

**Important**: Deploy in this order!

1. **Backend First** (Render)
   - Get backend URL
   - Update frontend environment

2. **Frontend Second** (Vercel)
   - Uses backend URL
   - Get frontend URL

3. **Update Backend CORS** (Render)
   - Add frontend URL to CORS
   - Add frontend URL to ALLOWED_HOSTS

## ⚠️ Common Pitfalls to Avoid

1. **Don't** deploy frontend before backend
2. **Don't** forget to update CORS after frontend deployment
3. **Don't** commit `.env` files to git
4. **Don't** use `DEBUG=True` in production
5. **Don't** forget to make `build.sh` executable
6. **Don't** use `http://` in production URLs (use `https://`)
